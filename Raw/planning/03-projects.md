## Visual Direction

- Call projects "Deployments" in nav/metadata, but keep "Projects" visible as a human title.
- Use large project panels, not tiny cards. Svarog Tracer should get the biggest visual treatment.
- Each project should have:
  - cinematic screenshot/video area
  - title and one-line value
  - stack tags
  - problem / solution / technical notes
  - demo and code links
- Consider one featured project full-width, then secondary projects below.

## Animation Ideas

- Scroll-triggered project reveal using clipping masks.
- Screenshot zooms from 1.08 to 1.0 as it enters.
- Stack tags appear like terminal packets.
- Project details can use a tab or segmented control: Overview, Stack, Build Notes.
- On hover, screenshot reveals a second layer: architecture diagram or feature labels.

## HR/Dev Clarity

Every project needs a plain-English outcome. Avoid only listing technologies.

Recommended formula:

> Built [product] that lets users [action], using [stack], with [important implementation].
