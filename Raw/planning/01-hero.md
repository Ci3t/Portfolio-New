# Section 01: Hero / Identity

## Purpose

Catch attention in the first 3 seconds and tell visitors exactly who Rani is.

## Main Message

Rani Ali is a full-stack developer with a UI-first eye, AI SaaS project experience, and strong communication from teaching developers.

## Content

- Name: RANI ALI
- Role line options:
  - Full Stack Developer
  - UI-Focused Full Stack Developer
  - Full Stack + AI Product Developer
  - Frontend-Leaning Full Stack Developer
- Short intro:
  - "I build polished full-stack products with sharp UI, reliable APIs, and AI-powered workflows."
  - "Full-stack developer focused on clean interfaces, practical AI tools, and production-ready web apps."
- Primary CTA: View Projects
- Secondary CTA: Contact / Open CV / GitHub
- Location should be real or abstract. Avoid fake coordinates.

## Visual Direction

- Keep the large RANI ALI wordmark.
- Keep the portrait frame, but use a real Rani image if available.
- Use sharp masks for the name and portrait reveal.
- Use technical metadata around the portrait: `SYS.PROFILE_01`, `STATUS: AVAILABLE`, `STACK: NEXT / NODE / AI`.
- Use a subtle grid and scanline texture, but keep the face/name readable.

## Animation Ideas

- Page load sequence:
  - tiny amber status lamp blinks
  - grid lines draw in
  - name reveals via vertical clip mask
  - role line slides in with character stagger
  - portrait wipes in from bottom with a rectangular mask
- Hover:
  - portrait switches from monochrome to color or higher contrast
  - CTA gets a sharp amber block shift
  - nav underline animates with a shared layout underline
- Exit to next section:
  - hero name scales slightly down and fades into grid
  - portrait mask closes as next section opens

## HR/Dev Clarity

The hero must not hide the role behind only cinematic language. Keep "Full Stack Developer" visible in plain English.
