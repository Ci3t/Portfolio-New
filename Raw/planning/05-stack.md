## Visual Direction

- Make this section feel like a command dashboard.
- Use grouped modules with sharp borders.
- Each skill should be represented as a compact chip, but important skill groups get short proof notes.
- Avoid overclaiming expertise. Group as "used in projects", "comfortable", or "learning next" if needed.

## Animation Ideas

- Skills stream in as rows of packets.
- Hovering a skill reveals where it was used.
- Background grid can pulse lightly when group changes.
- Use a filtering control: Frontend, Backend, AI, Product.

## HR/Dev Clarity

The stack section must include keywords for ATS/recruiters, but the project section must prove them.
